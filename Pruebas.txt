Cómo explicar el CRUD completo:

| Letra | Operación          | Método HTTP | Mongoose                               |
| ----- | ------------------ | ----------- | -------------------------------------- |
| C     | Create, crear      | POST        | `create()`                             |
| R     | Read, leer         | GET         | `find()`, `findOne()`, `findById()`    |
| U     | Update, actualizar | PUT o PATCH | `findByIdAndUpdate()`, `updateOne()`   |
| D     | Delete, eliminar   | DELETE      | `findByIdAndDelete()` o borrado lógico |

1. CREATE: crear un producto

Formulario del panel administrador
        ↓
handleSubmit
        ↓
createProduct
        ↓
POST /api/products
        ↓
protect
        ↓
adminOnly
        ↓
controlador createProduct
        ↓
Product.create
        ↓
MongoDB Atlas

“La creación comienza en AdminDashboard.jsx. El formulario ejecuta handleSubmit. Como no existe un editingId, la función llama a createProduct y envía los datos del formulario.”

“La creación real se realiza en product.controller.js. El controlador obtiene los datos desde req.body, valida la información, comprueba que no exista otro producto con el mismo código y utiliza Product.create para guardar el nuevo documento en MongoDB Atlas.”

“El Create comienza en el formulario de AdminDashboard.jsx. handleSubmit valida los campos y, como no existe editingId, llama a createProduct. En api.js se envía un POST a /api/products con JSON y token JWT. app.js dirige la solicitud a product.routes.js. La ruta pasa por protect y adminOnly, y luego ejecuta el controlador createProduct. El controlador valida los datos, comprueba que el código no esté repetido y usa Product.create para insertar el documento en MongoDB Atlas. Finalmente responde con código 201 y el frontend vuelve a cargar la tabla.”

2. READ:

“Cuando el usuario abre el catálogo, useEffect ejecuta la carga inicial. La respuesta del backend se guarda en el estado products y React genera una tarjeta para cada producto.”

“El Read del catálogo comienza en Catalog.jsx. Al cargar el componente, useEffect llama a getProducts. api.js envía un GET a /api/products. La ruta pública ejecuta getProducts y el controlador consulta MongoDB mediante Product.find({ activo: true }). La respuesta vuelve al frontend, se guarda con setProducts y React muestra las tarjetas.”

3. UPDATE:

En AdminDashboard: “Al presionar Editar, handleEdit guarda el ID del producto y carga sus datos en el formulario. Todavía no se actualiza la base de datos.”

LAS DOS FORMAS DE ACTUALIZAR SON: 

PUT: actualizar varios campos del producto.
PATCH: actualizar solamente el stock.

4. DELETE:

“La eliminación comienza en el botón Desactivar. Primero se pide confirmación y luego se envía el ID del producto al servicio.”

“El Delete del proyecto es un borrado lógico. En AdminDashboard.jsx, el botón Desactivar llama a handleDeactivate, pide confirmación y ejecuta deactivateProduct. api.js envía un DELETE a /api/products/:id. La ruta exige autenticación y rol administrador. El controlador utiliza Product.findByIdAndUpdate para cambiar activo a false. El documento no se elimina de MongoDB. Desaparece del catálogo porque la lectura pública solo busca productos con activo: true.”
